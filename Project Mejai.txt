SYSTEM MANIFEST: PROJECT MEJAI (V3 - Production Ready)
Version: 3.0.0 (Refined Logic & Edge Cases)
Target Audience: Lead Backend Engineers, Database Administrators, AI Prompt Engineers
Objective: To serve as the definitive, exhaustive technical and operational blueprint for "Project Mejai"—a highly scalable, AI-driven LINE Chatbot acting as a personal assistant, financial controller, fitness coach, and dynamically evolving anime roleplay companion.


1. COMPREHENSIVE SYSTEM ARCHITECTURE
The infrastructure is designed for high availability and asynchronous processing to prevent timeouts during peak LINE messaging traffic.
* Backend Framework: Node.js with TypeScript and Express.js. Deployed on a Serverless environment (e.g., Vercel or Google Cloud Functions).
* AI & Processing Brain: DeepSeek API. Uses deepseek-chat for routing/roleplay and Vision models for OCR.
* Database Layer: Supabase (Managed PostgreSQL) utilizing JSONB columns for flexible payload storage.
* Frontend Interface: LINE LIFF integrated with React.js and Tailwind CSS for the analytics dashboard and payment gateway (PromptPay QR via Omise/GB Prime Pay).


2. CORE FEATURES & LOGIC (THE 10 PILLARS - REFINED)
Pillar 1: Horoscope Engine
Refined Logic: AI generates horoscopes based on the user's birthdate stored in the DB. Triggers via direct user request or a daily automated cron job (Premium users only). Must include an astrological "lucky item" or "lucky color" for the day to encourage engagement.
Pillar 2: Secretary & Time Management
Edge Case & Logic: To resolve relative time ambiguity (e.g., "tomorrow at 2 PM"), the Node.js backend MUST inject the Server_Current_Timestamp (e.g., "Today is Monday, June 22, 2026, 10:41 PM +07:00") into the AI's System Prompt on every request. This ensures exact ISO 8601 date parsing for database storage and cron-job scheduling.
Pillar 3: Expense Controller
Fallback Logic: Extracts monetary values via Text/Vision. If the AI cannot confidently determine the expense category (e.g., user inputs "paid 500"), it must assign the category as uncategorized in the DB and proactively ask the user in character: "นายท่านจ่าย 500 บาทค่าอะไรไปหรอคะ? เค้าจะได้จดลงบัญชีถูกหมวดน้า~"
Pillar 4: Advanced Nutrition & Fitness Tracker
Strict Tracking Parameters: Focuses on macro calculations (Protein, Carbs, Fats) and micronutrients. Special Edge Case: The system actively aggregates daily Sodium intake. If the 24-hour rolling sodium intake exceeds 2,300 mg, the system triggers a Water_Retention_Alert, instructing the persona to warn the user about potential bloating and remind them to drink more water.
Pillar 5: 100% Anime Roleplay (Mejai Persona)
Strict Rule: The AI must NEVER break character. Context windows must carry the last 5-10 conversational turns (based on tier limit) to maintain narrative flow. The persona's tone is strictly governed by Pillar 9.
Pillar 6: Tiered Subscription Logic
Cost Control: Executed entirely via Middleware before the AI is invoked. If a Free tier user attempts to upload a receipt, the system immediately rejects the API call and responds with a pre-cached upsell message to preserve API tokens.
Pillar 7: DeepSeek API Integration (The Router Pattern)
Cost-Saving Edge Case: Do NOT send every message to the AI for intent classification. Implement a Regex / Keyword Pre-Processing Layer in Node.js. If a message exactly matches commands like /สรุปรายจ่าย, #ตารางงาน, or starts with predefined triggers, it routes directly to the Database service, bypassing the DeepSeek API entirely for intent checking.
Pillar 8: Vision & Emoji Parsing
Logic: Emojis in user inputs are passed directly to the AI as emotional context indicators. For Vision, receipts are processed to extract {total_amount, merchant_name, date} and food images extract {estimated_calories, estimated_protein}.
Pillar 9: Relationship State Machine & User Profiling
Explicit Scoring Rules (Affinity Score: 0-100):
* Positive Interaction (Compliments, sustained chat): +1 to +3 points.
* Negative Interaction (Abuse, harsh words): -5 points.
* Inactivity (No chat for >48 hours): -2 points per day.
* Anti-Spam Cap: Maximum gain of +10 points per 24-hour cycle to prevent users from manipulating the system.
Pillar 10: Analytics Dashboard (LIFF)
Logic: Data is fetched securely via JWT tokens linked to the user's LINE ID. Displays monthly expense breakdowns, daily protein/sodium progress bars, and the user's current Relationship Level.


3. DATABASE SCHEMA (SUPABASE POSTGRESQL)
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Core Users Table
CREATE TABLE users (
   id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
   line_user_id VARCHAR UNIQUE NOT NULL,
   display_name VARCHAR,
   subscription_tier VARCHAR DEFAULT 'free' CHECK (subscription_tier IN ('free', 'standard', 'premium')),
   message_count_today INT DEFAULT 0,
   subscription_expiry TIMESTAMP WITH TIME ZONE,
   created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
   updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Relationship & Persona Engine
CREATE TABLE user_relationships (
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   affinity_score INT DEFAULT 0 CHECK (affinity_score >= 0 AND affinity_score <= 100),
   relationship_status VARCHAR DEFAULT 'stranger',
   relationship_path VARCHAR DEFAULT 'neutral',
   personality_traits JSONB DEFAULT '{}'::jsonb,
   last_interacted_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
   PRIMARY KEY (user_id)
);

-- Data Lake for Analytics (Expenses, Nutrition, Schedule)
CREATE TABLE user_data_logs (
   id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   log_type VARCHAR NOT NULL CHECK (log_type IN ('expense', 'diet', 'schedule')),
   payload JSONB NOT NULL,
   logged_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
